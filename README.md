# Pixora

当前发布版本：`0.3.7`。`main` 分支可能包含尚未创建新 Release 的后续改进。

Pixora 是一个面向 Windows 10/11 x64 的 WPF 看图工具，支持图片/视频混合目录浏览、缩略图栏、快捷键、收藏、显示级旋转、排序切换、记住上次目录、裁剪、圆形裁剪、图片压缩、批量压缩、设置桌面壁纸、批量删除到回收站和视频封面另存。缩略图磁盘缓存支持设置容量上限、自动淘汰和手动清理。

主窗口可以记住大小、位置和最大化状态；再次从资源管理器双击图片时默认复用已有 Pixora 窗口。右侧缩略图栏保持固定宽度，可在单列和双列布局之间切换。

缩放时左上角会短暂显示当前比例，停止操作后自动淡出。设置中可以关闭提示，也可以选择两种不同含义：百分比以当前图片或安全预览的原始显示尺寸为基准，`100%` 表示该图像数据的原始显示尺寸；相对倍数以“适应窗口”的默认视图为基准，默认大小显示 `1.00×`，用于判断在初始视图上放大或缩小了多少。超大图片使用安全预览时，百分比不代表完整原图的像素比例。

图片信息会显示本次实际打开用时。直接从单文件启动时，Pixora 仍会在后台补齐同目录索引；快速完成的小目录索引不会用 `0.0 秒` 提示覆盖图片加载信息。

按 `Ctrl+K` 会在窗口中上方弹出快速搜索浮层。默认输入图片序号并按 Enter 跳转，也可以在设置中改为按文件名关键词查找；按 Esc 可随时收起。跳转成功后默认保留搜索框，便于连续查找；设置中可改为成功后自动收起。按住 Ctrl 后拖动搜索框可以调整位置，退出并重新打开 Pixora 会保留该位置；用快速搜索快捷键关闭后再打开会恢复默认位置。按文件名搜索且右侧缩略图栏可见时，缩略图栏会实时只显示匹配项；没有匹配项时保留右侧上一次有效内容和滚动位置，不显示空白栏。缩略图栏隐藏时不会为搜索额外构建或加载预览。

快速搜索刚打开且尚未选中输入框时，已经配置的查看器快捷键仍然优先执行，搜索框不显示文字光标，未绑定字符保持无动作；只有明确点击输入框后，文字和常用编辑键才会优先处理并显示正常闪烁光标，因此与快捷键相同的字母也能正常输入。点击输入框外的图片或缩略图区域会退出文字输入状态并恢复查看器快捷键。

手动打开过快速搜索后，Pixora 会默认在以后启动并完成目录加载后再次显示；可在设置中关闭“启动时自动显示快速搜索”。搜索结果中的长文件名会同时保留开头和结尾，完整名称可通过悬停查看。

设置窗口顶部提供常驻搜索，可筛选普通设置和快捷键。常规设置按文件行为、界面、性能与缓存、诊断与关于、文件关联分区显示。

大目录下，右侧缩略图只按需创建当前可见行和对应图片项，文件名搜索索引也只在首次实际搜索时建立。设置里的“缓存模式”可直接选择“自动（推荐）”或“手动”：自动模式按可用内存分档，并显示本机当前实际预算，16 GB 环境约为 `2 GB / 512 MB` 原图与预览缓存，32 GB 环境约 `4 GB / 1 GB`，64 GB 及以上约 `8 GB / 2 GB`；手动模式才启用两个容量下拉框。连续切图会优先预加载当前浏览方向，高配机器会适当增加邻图数量和缩略图解码并发；低内存保护仍会在系统繁忙时主动收缩。批量压缩的大量进度消息会合并刷新界面，这些优化不会减少可见功能或日志记录。

本项目由 AI 辅助编程完成。

## main 分支当前改进

- 新增超大 JPEG/PNG 安全预览和带内置缩略图或金字塔层的 BigTIFF 安全预览，避免直接完整解码数十亿像素主层。
- 新增真实图片打开耗时；快速完成的小目录索引不再显示容易误解的 `0.0 秒` 通知。
- 新增自动淡出的缩放比例提示，可在设置中选择绝对百分比或相对“适应窗口”的放大倍数。
- 增加超大图策略、BigTIFF IFD 选择、缩放格式和设置持久化回归测试。

## v0.3.7 主要更新

- 快速搜索跳转后默认继续显示，支持在设置中改为成功后自动收起；无文件名匹配时保留右侧上一次有效缩略图内容。
- 大目录缩略图改为按需创建并使用有界模型缓存，文件名索引延迟到首次实际搜索，减少启动时间和长期内存占用。
- 新增直观的“自动（推荐）/手动”缓存模式；自动模式展示本机实际预算，并为 16 GB、32 GB、64 GB 及以上环境自适应缓存、邻图预加载和缩略图并发。
- 目录文件大小和修改时间采用可失效的元数据快照，连续切图按浏览方向优先预加载，低内存时提前停止后台任务并收缩缓存。
- 快速搜索毛玻璃层复用渲染缓存，批量压缩进度合并刷新，同时保留完整处理日志；新增十五万项大目录无界面性能回归。

## 环境要求

运行发布包：

- Windows 10 x64（最低支持 22H2）或 Windows 11 x64
- .NET Desktop Runtime 9 x64

从源码构建：

- .NET 9 SDK

项目使用 WPF 和 `Magick.NET-Q16-AnyCPU`，目标框架为 `net9.0-windows`。

Windows 7、Windows 8.1 和 32 位 Windows 不在当前发布包的支持范围内，不建议下载或作为兼容性测试目标。

## 运行

```powershell
dotnet run --project src\Pixora\Pixora.csproj
```

也可以传入文件或文件夹路径：

```powershell
dotnet run --project src\Pixora\Pixora.csproj -- "D:\Pictures"
```

## 测试

```powershell
dotnet run --project tests\Pixora.SmokeTests\Pixora.SmokeTests.csproj
```

该 smoke test 会覆盖目录排序与取消、常见图片解码、GIF/HDR/AVIF、JPEG/PNG 超大图安全预览边界、BigTIFF 金字塔层选择、视频缩略图、媒体格式注册、内存与缩略图缓存、日志滚动、快捷键、设置持久化、缩放提示格式、裁剪数学、压缩和渲染控件，并包含 15 万项缩略图虚拟化、文件名索引和批量进度缓冲的无界面性能基准。

需要用本地超大图验证安全预览时，可以额外传入：

```powershell
dotnet run --project tests\Pixora.SmokeTests\Pixora.SmokeTests.csproj -- --large-image "D:\Samples\large-image.tif"
```

## 已知限制

- 批量压缩目前只处理静态图片；GIF、APNG、WebP 等动图会在预扫描后跳过，避免破坏动画帧。
- 视频文件只作为目录浏览项显示封面预览，不提供视频播放、裁剪或压缩。
- 视频封面来自 Windows Shell 缩略图；如果系统没有生成对应缩略图，Pixora 会显示内置占位封面。
- 缩略图磁盘缓存可在设置中开启，可选择容量上限；缓存达到上限后会按最近访问时间淘汰，设置页也可手动清理。首次超过上限后的少量新缩略图会在下一轮后台清理中回收。
- 快速滚动超大目录的缩略图栏时会暂停远离当前视口的解码，滚动停止后再恢复当前区域预加载，减少无效 CPU 和磁盘占用。
- 打开超大目录时会显示已发现项目数量，并可取消扫描；从单文件启动时会先显示当前图片，再在后台补齐目录。
- 当前目录被其他软件新增、重命名、修改或删除文件时，默认自动刷新列表，可在设置中关闭。
- 文件关联写入当前用户注册表项；如果移动了整个 Pixora 文件夹，先从新位置手动启动一次 Pixora，它会在不更改默认应用选择的前提下更新旧的程序路径。如果系统策略限制注册表写入，关联操作可能失败。
- 普通静态图片在 `120,000,000` 像素以内可以完整解码；超过后，JPEG 和 PNG 会使用 WIC 的原生目标尺寸解码生成超大图安全预览。普通性能档最长边为 `4096`，高性能档为 `8192`，Pixora 只在应用缓存中保留缩小后的位图；读取超大文件仍可能花费较长时间。
- 带内置缩略图或金字塔层的 BigTIFF 也支持安全预览：Pixora 按文件签名识别真实格式，即使扩展名错误也会选择最接近当前性能档、且在解码预算内的 IFD 层，不读取数十亿像素的主层。没有可用预览层的 BigTIFF 会提前拒绝。
- 安全预览会保留原图尺寸信息，但缩放上限是预览自身分辨率；裁剪、图片压缩和设置壁纸等必须读取全部原始像素的操作会被禁用。JPEG/PNG 直接安全预览的源图上限为 `2,000,000,000` 像素；BMP、WebP、HEIF/AVIF 等当前未验证为原生缩放解码的超大格式仍会在完整像素解码前拒绝加载。
- HDR 和动图继续使用各自的保护阈值。BigTIFF 当前只使用已有金字塔层生成整图预览，不提供按瓦片读取原始分辨率和任意级别深度缩放，因此仍不是专业科研影像或病理切片查看器。

## 批量压缩

软件右键菜单里的“批量压缩图片”会打开可视化批量压缩窗口，支持选择文件/目录、输出目录、质量、最大宽高、格式、子目录和取消任务；窗口大小、位置和最大化状态会自动记住。

设置页里可以开启“启动时自动打开上次目录”。即使不开启，手动“打开目录”也会默认从上次成功打开的位置开始。

`tools` 目录里的脚本仍保留作为备用：

```text
tools\batch-compress.ps1
```

脚本详细用法见：

```text
tools\batch-compress-README.md
```

脚本依赖 ImageMagick CLI 的 `magick` 命令。常用示例：

```powershell
.\tools\batch-compress.ps1 -InputPath "D:\Pictures" -Quality 82 -MaxWidth 1920 -MaxHeight 1920 -Format jpg
```

## 主要目录

- `src\Pixora`：主程序源码。
- `src\Pixora\Services`：图片加载、压缩、目录索引、快捷键、设置和日志等服务。
- `src\Pixora\Controls`：自定义图片渲染控件。
- `tests\Pixora.SmokeTests`：控制台式冒烟测试。
- `test-images`：测试样本图片。
- `tools`：外部辅助脚本。

## 文件关联和默认应用

设置页里的“关联常见图片格式”会把 Pixora 加入 Windows 的“打开方式”列表。“关联并设为默认应用”会先尝试官方静默设置；当前 Windows 不支持时，会打开 Pixora 的默认应用设置页让用户确认。

“高级一键默认”需要用户自行提供外部工具 `SetUserFTA.exe`，并放到发布目录的 `tools` 文件夹中。该模式会在明确确认后批量执行外部工具，把支持的图片扩展名设为 `Pixora.Image`。这是非官方高级方案，Pixora 不内置该工具。

## 本地数据

设置和错误日志写入：

```text
%LOCALAPPDATA%\Pixora
```

其中包括 `viewer-settings.json`、`shortcuts.json`、`favorites.json` 和 `error.log`。旧版 `%LOCALAPPDATA%\Pic-O` 和更早的 `%LOCALAPPDATA%\PureView` 中的设置、快捷键、收藏和批量压缩设置会在首次启动时迁移。

“复制诊断信息”默认隐藏本机完整路径，只保留排查问题需要的版本、系统、缓存和当前状态信息。设置中可以临时允许包含完整路径。
